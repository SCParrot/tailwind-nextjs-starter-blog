# 介绍

Markdown 和 Mdx 解析通过 `unified` 以及其他 remark 和 rehype 包支持。`next-mdx-remote` 允许我们以更灵活的方式解析 `.mdx` 和 `.md` 文件，无需修改 webpack。

使用的是 GitHub 风格的 Markdown。`mdx-prism` 为代码块提供语法高亮功能。以下是所有功能的演示。

以下 Markdown 速查表改编自：https://guides.github.com/features/mastering-markdown/

# 什么是 Markdown？

Markdown 是一种在网络上设置文本样式的方式。您可以控制文档的显示方式；将单词设置为粗体或斜体、添加图片以及创建列表只是我们可以使用 Markdown 完成的几件事。大多数情况下，Markdown 只是普通文本，只是添加了一些非字母字符，如 `#` 或 `*`。

# 语法指南

以下是您可以在 GitHub.com 上的任何地方或自己的文本文件中使用的 Markdown 语法概述。


## 表格

您可以通过组装单词列表并用连字符 `-`（第一行）分隔它们，然后用竖线 `|` 分隔每一列来创建表格：

```
| 第一列标题                | 第二列标题                |
| --------------------------- | ---------------------------- |
| 单元格 1 的内容         | 单元格 2 的内容          |
| 第一列的内容 | 第二列的内容 |
```

| 第一列标题                | 第二列标题                |
| --------------------------- | ---------------------------- |
| 单元格 1 的内容         | 单元格 2 的内容          |
| 第一列的内容 | 第二列的内容 |

## 删除线

任何用两个波浪号包裹的单词（如 `~~this~~`）都会显示为 ~~删除线~~。
