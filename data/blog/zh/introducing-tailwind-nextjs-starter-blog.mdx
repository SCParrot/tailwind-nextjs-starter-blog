# Tailwind Nextjs 启动博客

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/git/external?repository-url=https://github.com/timlrx/tailwind-nextjs-starter-blog)

> [!CAUTION]
> 这是版本 1 模板的 README。出于历史原因保留，不再受支持。它也很好地展示了 [Github Alert](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax#alerts)。有关实际文档，请参阅 [Github 仓库](https://github.com/timlrx/tailwind-nextjs-starter-blog)。

这是 [Next.js](https://nextjs.org/)、[Tailwind CSS](https://tailwindcss.com/) 博客启动模板。可能是功能最丰富的 Next.js markdown 博客模板之一。开箱即用，配置了最新技术，让技术写作变得轻松。易于配置和自定义。是现有 Jekyll 和 Hugo 个人博客的完美替代品。

查看下面的文档以开始使用。

## 动机

我想将现有博客移植到 Nextjs 和 Tailwind CSS，但没有现成的开箱即用模板可以使用，所以我决定创建一个。设计借鉴自 [Tailwindlabs 博客](https://github.com/tailwindlabs/blog.tailwindcss.com)。

我希望它几乎与流行的博客模板（如 [beautiful-jekyll](https://github.com/daattali/beautiful-jekyll) 和 [Hugo Academic](https://github.com/wowchemy/wowchemy-hugo-modules)）一样功能丰富，但具有 React 生态系统和当前 Web 开发最佳实践的最佳特性。

## 特性

- 使用 [Tailwind 3.0](https://tailwindcss.com/blog/tailwindcss-v3) 和主色属性轻松自定义样式
- 几乎完美的 lighthouse 分数 - [Lighthouse 报告](https://www.webpagetest.org/result/210111_DiC1_08f3670c3430bf4a9b76fc3b927716c5/)
- 轻量级，45kB 首次加载 JS，在生产构建中使用 Preact
- 移动友好视图
- 浅色和深色主题
- 使用 [Fontsource](https://fontsource.org/) 自托管字体
- 支持 [plausible](https://plausible.io/)、[simple analytics](https://simpleanalytics.com/) 和 google analytics
- [MDX - 在 markdown 文档中编写 JSX！](https://mdxjs.com/)
- 通过 [rehype-prism-plus](https://github.com/timlrx/rehype-prism-plus) 进行服务器端语法高亮显示，带有行号和行高亮
- 通过 [KaTeX](https://katex.org/) 支持数学显示
- 通过 [rehype-citation](https://github.com/timlrx/rehype-citation) 支持引用和参考文献
- 通过 [next/image](https://nextjs.org/docs/basic-features/image-optimization) 进行自动图像优化
- 通过 [mdx-bundler](https://github.com/kentcdodds/mdx-bundler) 进行灵活的数据检索
- 支持标签 - 每个唯一标签将是自己的页面
- 支持多位作者
- 博客模板
- TOC 组件
- 支持博客文章的嵌套路由
- 邮件订阅组件，支持 Mailchimp、Buttondown、Convertkit、Klaviyo、Revue、Emailoctopus 和 Beehiiv
- 支持 [giscus](https://github.com/laymonage/giscus)、[utterances](https://github.com/utterance/utterances) 或 disqus
- 项目页面
- 预配置的安全头
- SEO 友好，具有 RSS 订阅、站点地图等！

## 快速入门指南

1. JS（官方支持）- `npx degit https://github.com/timlrx/tailwind-nextjs-starter-blog.git` 或 TS（社区支持）- `npx degit timlrx/tailwind-nextjs-starter-blog#typescript`
2. 个性化 `siteMetadata.js`（站点相关信息）
3. 如果您想使用任何分析提供商或除 giscus 之外的评论解决方案，请修改 `next.config.js` 中的内容安全策略
4. 个性化 `authors/default.md`（主要作者）
5. 修改 `projectsData.ts`
6. 修改 `headerNavLinks.ts` 以自定义导航链接
7. 添加博客文章
8. 部署到 Vercel

## 开发

首先，运行开发服务器：

```bash
npm start
# 或
npm run dev
```

在浏览器中打开 [http://localhost:3000](http://localhost:3000) 查看结果。

您可以通过修改 `pages/index.js` 开始编辑页面。当您编辑文件时，页面会自动更新。

## 扩展/自定义

`data/siteMetadata.js` - 包含大部分站点相关信息，应根据用户需求进行修改。

`data/authors/default.md` - 默认作者信息（必需）。可以在 `data/authors` 中添加额外作者。

`data/projectsData.js` - 用于在项目页面上生成样式卡片的数据。

`data/headerNavLinks.js` - 导航链接。

`data/logo.svg` - 替换为您的自己的徽标。

`data/blog` - 替换为您的自己的博客文章。

`public/static` - 存储图像和图标等资源。

`tailwind.config.js` 和 `css/tailwind.css` - 包含可以修改以更改站点整体外观的 tailwind 样式表。

`css/prism.css` - 控制与代码块相关的样式。请随意自定义并使用您喜欢的 prismjs 主题，例如 [prism themes](https://github.com/PrismJS/prism-themes)。

`components/social-icons` - 要添加其他图标，只需从 [Simple Icons](https://simpleicons.org/) 复制一个 svg 文件并在 `index.js` 中映射它们。其他图标使用 [heroicons](https://heroicons.com/)。

`components/MDXComponents.js` - 通过在此处指定来传递您自己的 JSX 代码或 React 组件。然后可以在 `.mdx` 或 `.md` 文件中直接调用它们。默认情况下，传递了自定义链接和图像组件。

`layouts` - 页面中使用的主要模板。

`pages` - 要路由到的页面。阅读 [Next.js 文档](https://nextjs.org/docs) 以获取更多信息。

`next.config.js` - 与 Next.js 相关的配置。如果要从其他域加载脚本、图像等，需要调整内容安全策略。

## 文章

### Frontmatter

Frontmatter 遵循 [Hugo 的标准](https://gohugo.io/content-management/front-matter/)。

目前支持 10 个字段。

```
title（必需）
date（必需）
tags（必需，可以是空数组）
lastmod（可选）
draft（可选）
summary（可选）
images（可选，如果没有提供，默认为 siteMetadata 配置中的 socialBanner）
authors（可选列表，应与 `data/authors` 中的文件名对应。如果没有指定，则使用 `default`）
layout（可选列表，应与 `data/layouts` 中的文件名对应）
canonicalUrl（可选，文章的规范 URL，用于 SEO）
```

这是文章 frontmatter 的示例：

```
---
title: '介绍 Tailwind Nexjs 启动博客'
date: '2021-01-12'
lastmod: '2021-01-18'
tags: ['next-js', 'tailwind', 'guide']
draft: false
summary: '寻找一个高性能、开箱即用的模板，具有网络技术中最好的功能来支持您的博客需求？查看 Tailwind Nextjs 启动博客模板。'
images: ['/static/images/canada/mountains.jpg', '/static/images/canada/toronto.jpg']
authors: ['default', 'sparrowhawk']
layout: PostLayout
canonicalUrl: https://tailwind-nextjs-starter-blog.vercel.app/blog/introducing-tailwind-nextjs-starter-blog
---
```

### 编写

运行 `node ./scripts/compose.js` 来引导新文章。

按照交互式提示生成具有预填充 front matter 的文章。

## 部署

**Vercel**  
部署模板的最简单方法是使用 [Vercel 平台](https://vercel.com)，Next.js 的创建者。查看 [Next.js 部署文档](https://nextjs.org/docs/deployment) 了解更多信息。

**Netlify / GitHub Pages / Firebase 等**  
由于模板使用 `next/image` 进行图像优化，必须进行额外配置才能部署到其他流行的静态托管网站，如 [Netlify](https://www.netlify.com/) 或 [GitHub Pages](https://pages.github.com/)。必须使用替代的图像优化提供商，如 Imgix、Cloudinary 或 Akamai。或者，将 `next/image` 组件替换为标准的 `<img>` 标签。查看 [`next/image` 文档](https://nextjs.org/docs/basic-features/image-optimization) 了解更多信息。

邮件订阅组件中使用的 API 路由不能在静态站点导出中使用。您需要使用表单 API 端点提供商并在邮件订阅组件中相应替换路由。其他托管平台如 Netlify 也提供替代解决方案 - 请参阅其文档了解更多信息。
