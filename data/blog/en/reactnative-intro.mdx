In mobile app development, "develop once, run on multiple platforms" has always been the goal pursued by developers. **React Native** was born under this backdrop ‚Äî it allows us to use **JavaScript / TypeScript and React concepts** to build truly native-like iOS and Android apps.

## What is React Native?

**React Native (RN)** is an open-source cross-platform mobile development framework launched by Facebook (Meta).
It doesn't simply wrap web pages in a WebView, but instead uses a **native component mapping mechanism** to render React components as genuine native UI.

This is also the biggest difference between React Native and traditional Hybrid solutions.

## How React Native Works üß†

React Native doesn't directly manipulate native UI, but instead follows this process:

1. JavaScript code runs in JS engine (Hermes / JSC)
2. React generates Virtual DOM
3. RN framework passes updates through Bridge (or new architecture) to native layer
4. Native layer renders corresponding UI components

With the introduction of the **New Architecture (Fabric + TurboModules)**, React Native has seen significant improvements in performance and communication efficiency.

## A Simple React Native Example

```tsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.text}>Hello React Native üöÄ</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
  },
  text: {
    fontSize: 20,
  },
});
```

From the code, you can see that React Native is very similar to React, but uses **cross-platform native components and styling systems**.
